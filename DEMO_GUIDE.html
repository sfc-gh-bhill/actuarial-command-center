Pre-Demo Checklist

5 Minutes Before — Environment

Open Streamlit app in Snowsight: ACTUARIAL_DEMO.GOLD.ACTUARIAL_COMMAND_CENTER
Click through all 9 pages once to pre-warm caches (prevents loading spinners during demo)
On the Home page, set the Audience toggle to "Executive"
Navigate back to the Home page (Mission Control) — this is your starting position
Verify the proactive alert banner (TX BH cost spike) is visible on the Home page
On the Intelligence Agent page, clear any previous chat messages
Optionally open Snowsight in a second browser tab for showing raw SQL if asked

Browser Setup

Use Chrome or Edge (best Streamlit rendering)
Set browser zoom to 90% so full page fits without scrolling
Close all other tabs and notifications
If screen sharing: share the specific browser window, not entire screen

Demo Mode (No Snowflake Connection)

The app works fully in Demo Mode using synthetic fallback data. If you cannot connect to Snowflake:

Run locally: streamlit run streamlit_app.py
The connection status badge will show "Demo Mode (Local)" instead of "Snowflake Connected"
All pages, charts, KPIs, and the Intelligence Agent work identically in demo mode
The Contract Repricing stored procedure call is simulated locally with the same math

Step 1: Mission Control & Executive Summary 0:00 – 3:00

Pages: Home (Mission Control) → Executive Summary  |  Goal: Anchor the narrative, establish the problem

Home Page — Mission Control

"Today I'm going to show you an actuarial command center built entirely on Snowflake. Everything you see—the data pipeline, the ML models, the dashboards, and the AI agent—runs inside the Snowflake AI Data Cloud. No data leaves the platform."
1
Point to the 4 KPI tiles at the top of Mission Control
Health Plan Co.l Loss Ratio, Margin, Claims Processed (~800K), Cost PMPM. Say: "These are live from a governed Semantic View—the single source of truth."
2
Point to the orange/red proactive alert banner
It reads: TX Behavioral Health cost spike detected. Say: "Before we even started looking, the system flagged something. Cortex ML is running anomaly detection continuously. We're going to follow this thread."
3
Briefly point to the Quick Actions row (Draft CFO Briefing, Audit Package, Run Repricing)
Say: "Every section of this app has actions—not just charts to look at. You can draft emails, build audit packages, or run scenarios."

Executive Summary Page

4
Click "Executive Summary" in the left nav blade (or use the demo workflow tiles)
This page opens with enterprise-wide financial KPIs.
"This is the CFO's Monday morning view. Health Plan Co.l Loss Ratio across the book, margin by line of business, anomaly alerts, and trend charts—all governed by the same Semantic View the AI agent uses."
5
Point to the MLR by LOB tiles
Highlight any LOB near or above the ACA threshold. Say: "ACA requires 80% MLR for individual and small group, 85% for large group. Below that, we owe rebates. Above that, we're watching margin."
6
Click "Email Summary" in the action bar
Show the pre-drafted email with data-backed insights. Say: "One click—a Cortex-drafted executive briefing ready to send." Close the composer after a brief pause.
Anchor the demo: the audience should now understand this is a real working application with governed data, not a set of mockup slides. The TX BH alert is the hook—carry it forward.
Do NOT spend time on the Architecture page at this point. If asked, say "I'll show you that at the end" or briefly toggle to Technical mode to show the pipeline diagram.

Step 2: Trend Surveillance — The Early Warning 3:00 – 6:00

Page: Trend Surveillance  |  Goal: Show Cortex ML anomaly detection, build urgency
"We saw the alert on the home page. Let's dig into what the system actually found."
1
Click "Trend Surveillance" in the nav blade
The page opens with Cortex ML anomaly alerts at the top and cost trend charts below.
2
Point to the anomaly alert card for TX Behavioral Health

3

4
Show the Texas Behavioral Health deep dive section
Root cause analysis with cost drivers. Say: "The system doesn't just flag the problem—it shows you what's driving it."
5
(Optional, if time) Click "Create Alert Rule" to show the rule builder UI

"So now we know: Texas behavioral health costs are running hot. The question is—what can we do about it? That's where contract repricing comes in."
This is the narrative bridge. The anomaly was detected by Cortex ML, not by a human looking at a report. Emphasize the proactive, continuous nature.

Step 3: Contract Repricing — The Wow Moment 6:00 – 10:00

Page: Contract Repricing  |  Goal: Interactive "what-if" that gets a reaction
"We found the spike. Now let's model what happens if we renegotiate the Texas provider contract. This runs a Python stored procedure live inside Snowflake."
1
Click "Contract Repricing" in the nav blade
The page loads with the repricing controls on the left and results on the right.
2
Set Target State to "TX" (Texas)
The state selector should already default to TX or be easily selectable.
3
Slide the Basis Point Adjustment slider to -200 BPS
Say: "A negative 200 basis point adjustment means we're asking for a 2% reduction in reimbursement rates. Watch the before-and-after KPIs update." Point to: current vs. repriced paid amount, MLR impact, and margin change.
4
Point to the IP/OP pass-through percentages
Say: "Inpatient gets 100% of the rate change, outpatient gets 50%. This mirrors how real contract amendments work—you negotiate different rates for different service types."
5
Click "Save Scenario" to store this as Scenario 1
Say: "Save this scenario, then model a second one." Slide BPS to -350, save as Scenario 2. Point to the side-by-side comparison.
6
Click "Negotiation Brief" in the action bar
Show the mock negotiation document. Say: "This pulls actual contract clauses from Cortex Search and combines them with the repricing data. Your network team walks into the negotiation with numbers AND contract language."
"This is the moment in the demo where heads nod. Every health plan does this in Excel with stale data. Here it's live, on real claim volume, with the contract terms right there."
The "wow" is the interactivity and speed. Slide, see results, compare scenarios, generate a brief—all in seconds. Do NOT rush through this. Let the audience absorb the before/after numbers.
Avoid going deeper than 2 scenarios. The side-by-side comparison is the payoff. More than 2 scenarios dilutes the impact.
🤖

Step 4: Intelligence Agent — The CFO Bridge 10:00 – 13:00

Page: Intelligence Agent  |  Goal: Show the Cortex Agent combining structured data + unstructured documents
"Now imagine it's Thursday afternoon and the CFO pings you: 'Why is Texas underperforming and what's our exposure?' Instead of pulling data from three systems and writing a memo, you just ask."
1
Click "Intelligence Agent" in the nav blade
The chat interface opens with suggested questions in the sidebar and workflow templates.
2
Type (or click the suggested question): "Why is our Texas margin missing the target, and what is our exposure?"
This is the primary demo question. It triggers both agent tools: the Semantic View for financial data and Cortex Search for contract documents.
3
Wait for the response, then walk through the answer
Point out these elements in the response:
• Financial data: MLR, margin numbers from the Semantic View (same numbers as the Executive Summary page)
• Contract clauses: TX provider contract terms pulled from Cortex Search
• Synthesized recommendation: The agent combines both into an actionable answer
"Look at what just happened. The agent searched actual provider contract documents stored in Snowflake and pulled back specific reimbursement rates. It didn't hallucinate them. It cited the source. This is RAG running entirely inside Snowflake. And the financial data came from the same governed Semantic View that powers the dashboards."
4
Click "Email This" on the agent response
Show the email composer with the agent's analysis pre-filled. Say: "One click to turn that into a CFO briefing."
5
(Optional, if time) Ask a follow-up: "What are the behavioral health reimbursement rates in the Texas contract?"
This isolates the Cortex Search tool and shows contract RAG specifically. Good for technical audiences.
The dual-tool architecture (Semantic View + Cortex Search) is the key differentiator. Competitors offer BI dashboards OR document Q&A. Snowflake's Cortex Agent does both from one governed platform in a single answer.

Objection Handling

ObjectionResponse
"This is synthetic data" "By design for compliance. The distributions are actuarially accurate—Gamma for outpatient, Lognormal for inpatient, Box-Muller transforms. We use APCD-CDL v3.0.1 standards. In a POC we'd replicate this with your actual claims data."
"We already have a BI tool" "This doesn't replace BI. The Semantic View means Tableau, Power BI, or Sigma all get the same governed truth. What BI can't do is the AI agent, anomaly detection, and one-click actions you just saw."
"How does the AI agent avoid hallucination?" "Two safeguards: (1) the Semantic View constrains it to governed metrics with defined calculations, and (2) Cortex Search returns actual document chunks with citations, not generated text. The agent reasons over real data."
"What about HIPAA / data governance?" "Data never leaves the Snowflake security perimeter. The Streamlit app, the ML models, and the AI agent all run inside Snowflake. Role-based access, column masking, and audit trails are built in."
"Can this scale to our volume?" "This demo runs 800K claims on an X-Small warehouse. Snowflake's elastic compute means you scale the warehouse, not the architecture. Dynamic Tables handle the pipeline. Same code, any volume."
"How long to build this?" "The foundation—data model, dynamic tables, semantic view—is days. The Streamlit app is a week. The AI agent is hours once the Semantic View and Cortex Search service exist. The hard part is your data, not our platform."
